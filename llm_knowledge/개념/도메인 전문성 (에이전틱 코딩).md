# 도메인 전문성 (에이전틱 코딩)

Anthropic [[Claude Code]] 40만 세션 연구의 핵심 변수 — **과제별(task-specific)** 이해 깊이. 코딩 **기술**이 아니라 **풀려는 문제**에 대한 숙련.

## 정의 (5단계)

Clio/분류기 신호: (1) 지시 정밀도 (2) 검증 요청 (3) 사용자↔Claude **교정 방향**

| 예 | 수준 |
|----|------|
| Rust 처음인 시니어 SWE | Rust 과제 = **novice** |
| Python 미사용 회계사가 정산 규칙·예외를 정밀 지시 | 그 과제 = **expert** |

## 성공과의 관계

- Verified success: novice **15%** → intermediate+ **28–33%**  
- **대부분의 이득은 novice→intermediate**; mastery 추가분은 작음  
- *Competence, not mastery* — 작동 수준의 domain grasp가 대부분 포착

## 에이전트 활용량

전문가일수록 프롬프트당 Claude **actions·output** 증가 → 한 번에 더 멀리 위임 가능(검증·함정 인지).

## amplify vs substitute

| | 역할 |
|--|------|
| **Substitute** | 범용 코딩 구현 — 직군 간 성공률 수렴 |
| **Amplify** | 도메인 판단·검증·steering — 전문성이 성공·회복력 좌우 |

> *Coding agents are not substituting for domain expertise.*
