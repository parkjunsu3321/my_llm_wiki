# 바이브 코딩

**Vibe coding** — 구조화된 명세 없이 **프롬프트·대화 흐름(vibe)** 만으로 AI에게 코딩을 맡기는 방식. [[에이전틱 코딩]]의 비공식·저구조 변형으로 자주 대비됨.

## 특징

- 맥락 **최소화** — "인벤토리 만들어줘", "버그 고쳐줘" 등 즉석 지시
- **소규모·프로토타입**에 강함 ([[출처/스펙 주도 개발 SDD tsyang Context Engineering]])

## 문제 (SDD 도입 동기)

- 대화가 길어질수록 AI가 **이전 패턴·스타일을 잊음**
- **[[Context Rot]]** — 작업 전환 시 맥락 부식, 독립 코드 조각들의 모음
- **컨텍스트 노동** — 맥락을 수동 반복 입력
- 팀 **일관성·추적** 어려움 ([[출처/iFKakao Agentic Coding 카카오페이 Spec-kit]])

- [[루프 엔지니어링]]: 반복·메모리로 맥락 유지
