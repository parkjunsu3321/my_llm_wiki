# 컨텍스트 엔지니어링

**Context Engineering** — AI에게 **어떤 맥락을·얼마나 구조화해 전달할지** 설계하는 일. [[Claude 프롬프트 엔지니어링]]과 구분:

| | 프롬프트 엔지니어링 | 컨텍스트 엔지니어링 |
|--|---------------------|---------------------|
| 초점 | **어떻게** 말하느냐 | **무엇을** 알려주느냐 |
| 예 | 지시 형식·역할·톤 | 아키텍처·컨벤션·요구사항·제약 |

같은 모델도 맥락 품질에 따라 출력이 크게 달라짐 ([[출처/스펙 주도 개발 SDD tsyang Context Engineering]]).

## 맥락 전달 방식 스펙트럼

1. **전통 개발** — 맥락이 사람·구두·낡은 문서에 분산 (버스 팩터)
2. **정적 가이드** — `CLAUDE.md`, `AGENTS.md`, `.cursorrules` (전역, 기능 단위·게이트 약함)
3. **선택적 라우팅** — [[하이브리드 컨텍스트 라우팅]]·[[token-router]]: 로컬 triage + **원문 slice** (요약 대신) ([[출처/token-router 하이브리드 컨텍스트 라우터 GeekNews]])
4. **[[바이브 코딩]]** — 맥락 최소 → [[Context Rot]]
5. **[[스펙 주도 개발]]** — **스펙 = 구조화된 맥락**; 작업 전환해도 동일 spec 참조

## SDD와의 관계

> 스펙을 잘 쓰는 것 = 컨텍스트 엔지니어링을 잘 하는 것

- Spec에 원칙·아키텍처·요구·테스트 기준 통합
- 코드는 spec의 표현; **좋은 맥락 → 좋은 코드**
