# 자기 갱신 위키

**Self-rewriting knowledge base** — 새 정보가 들어올 때 페이지를 **덧붙이지 않고 기존 노트를 재작성**해 일관성을 유지하는 지식 관리 방식. [[obsidian-second-brain]]의 핵심 철학.

## append-only의 한계

- 노트만 쌓이면 **오래된 사실 + 새 사실** 공존
- 모순은 **사람이 수동** 정리
- 규모↑ → 검색·신뢰도↓

## 자기 갱신 접근

| 메커니즘 | 역할 |
|----------|------|
| **재작성** | ingest 시 새 페이지 대신 **기존 페이지 갱신** (인물·주장·사실) |
| `/obsidian-reconcile` | 모순 **자동 조정** |
| `/obsidian-synthesize` | 출처 간 **패턴 → 종합 페이지** |
| **예약 에이전트** | 야간 마감·주간 리뷰·모순 점검·고아 노트·인덱스 재생성 |

## [[LLM Wiki]]와의 관계

Karpathy [[LLM Wiki]]는 출처마다 **새 위키 페이지 + cross-link** (append 성향). 자기 갱신 위키는 **동일 개념·엔티티 페이지를 living document**로 취급.
