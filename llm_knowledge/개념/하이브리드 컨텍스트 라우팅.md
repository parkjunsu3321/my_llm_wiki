# 하이브리드 컨텍스트 라우팅

**Hybrid context routing** — 대용량 로그·코드·에이전트 지침을 클라우드 LLM에 통째로 넣지 않고, **로컬 소형 모델로 좌표 탐색 → 원문 slice → 클라우드 추론**으로 나누는 [[컨텍스트 엔지니어링]] 패턴.

## 원칙

> **탐색은 로컬(무료·빠름), 추론은 클라우드(고성능)** — 단, **요약하지 않고 원문 slice**

- 소형 모델 **요약**은 누락·왜곡 위험 → [[Context Rot]]·환각 유발
- slice는 디스크 **raw text** — 클라우드 추론 능력은 유지하면서 **입력 토큰**만 절감

## 3단계 파이프라인

| 단계 | 담당 | 출력 |
|------|------|------|
| Local Triage | Ollama SLM (예: Gemma 4 2B) | 관련 **라인 번호** |
| Raw Slicing | 스크립트 | 원문 텍스트 조각 |
| Cloud Reasoning | Codex·Claude Code 등 | 디버깅·코드 작성 |

## 적용 대상

- **error_log** — CI·인프라 대용량 로그
- **heavy_code** — 레거시·대형 파일
- **agent_context** — 긴 `CLAUDE.md`·`AGENTS.md`·`.cursorrules`·reference MD에서 **작업 관련 구간만** ([[출처/token-router 하이브리드 컨텍스트 라우터 GeekNews]])

Root 에이전트 지침은 보통 **자동 전량 주입** → 사후 라우팅 불가. **짧은 root + reference 분리**가 전제.

## 한계·운영

- 초소형(2B) 모델은 JSON 좌표 출력 **문법 오류** 가능 → 4B·7B coder 모델로 교체 (`OLLAMA_MODEL`)
- slice가 좁으면 **역요청**으로 범위 확장 (안전장치)
- triage 후 **VRAM 해제**로 로컬 부담 완화
