# 에이전틱 코딩

**Agentic coding** — AI 에이전트가 프롬프트·도구 호출로 **자율적으로 코드를 생성·수정**하는 개발 방식. 생산성은 크지만 **통제·일관성** 문제가 프로젝트가 커질수록 두드러짐.

## 특징

- 개발자: 프롬프트·지시·검토 (**지휘자**)
- AI: 코드 생성, MCP·빌드·테스트 등 도구 사용
- 초·중·후기 **단계별 협업 전략**으로 환각 완화 시도 ([[출처/iFKakao Agentic Coding 카카오페이 Spec-kit]])

## 한계 (엔터프라이즈 관찰)

| 문제 | 설명 |
|------|------|
| **무분별한 생성** | 후반으로 갈수록 여기저기 코드 증가 |
| **환각** | 잘못된 로직·API·패턴 |
| **검토 부담** | AI 코드 수정·검토 시간 > 직접 코딩 |
| **맥락 상실** | 긴 대화·[[바이브 코딩]]에서 패턴·스타일 불일치 |

→ 업계·[[카카오페이 AI 플랫폼]] 사례: **[[스펙 주도 개발]]**·[[Spec-kit]] 등 **스펙 선행 + 단계별 수행**으로 전환

## SDD와의 관계

- 에이전틱 코딩: **실행 중심** (프롬프트 → 코드)
- SDD: **명세 중심** (constitution·spec → plan·tasks → implement)
- 역할 이동: 코드 작성 → **스펙 검증·테스트 설계** ([[출처/iFKakao Agentic Coding 카카오페이 Spec-kit]])

## Anthropic 40만 세션 (Claude Code)

[[Claude Code]] 대화형 세션 분석 — **사람=plan ~70%**, **에이전트=exec ~80%**. 성공은 코딩 직군보다 **과제별 [[도메인 전문성 (에이전틱 코딩)|도메인 전문성]]**; verified success novice **15%** vs expert **33%**. 직군 간 격차 **~5%p**. *Coding is a leading case* for other knowledge work.
