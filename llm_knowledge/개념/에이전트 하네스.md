# 에이전트 하네스

기반 LLM의 원시 출력을 **구조화된 행동**으로 바꾸는 **실행 환경·인터페이스** — 프롬프트, 도구, 메모리, 제어 흐름, 샌드박스, 관측.

[[루프 엔지니어링]]의 개념 층: 프롬프트 → 컨텍스트 → **harness** → 루프.

## HarnessX 정의 ([[HarnessX]])

$\mathcal{H}=(\mathcal{M},\mathcal{C})$

| 구성 | 내용 |
|------|------|
| $\mathcal{M}$ | 모델·역할·fallback |
| $\mathcal{C}=(\mathbf{P},\mathbf{S})$ | Processor 파이프라인 + 공유 슬롯(도구·트레이서·워크스페이스 등) |

**1급 객체** — 직렬화·비교·해시·치환 가능 → 프로그램적 진화 전제.

## Processor

hook당 `process(event) → events`: pass-through · transform · split · intercept · interrupt.

## 9차원 (HarnessX)

D1 모델 · D2 컨텍스트 · D3 메모리 · D4 도구 · D5 실행환경 · D6 평가 · D7 제어·안전 · D8 관측 · D9 학습 브릿지.

## 기존 접근 vs HarnessX

| 유형 | 예 | 한계 |
|------|-----|------|
| 부품 라이브러리 | LangChain, LlamaIndex | 조합은 앱 코드 |
| 오케스트레이션 | LangGraph, AutoGen, CrewAI | 패턴 조합 수동 |
| 제품 harness | Claude Code, Cursor | 구조 고정, 수동 갱신 |
| **Foundry** | [[HarnessX]] | 트레이스 기반 compose·adapt·evolve |
